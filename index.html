Examples: qwen3.5-0.8b-mlx:3 for LM Studio; google/gemini-2.5-flash for Vertex AI; meta-llama/llama-3.3-70b-instruct:free for OpenRouter.

Vertex AI uses Google Cloud OAuth access tokens, not normal Gemini API keys. Run gcloud auth print-access-token and paste the result into API Key. The token expires, so this is for testing only.

Try global first. If it fails, try us-central1.

Vertex AI testing
  1. In Terminal, run: gcloud config get-value project
  2. If needed, set the project: gcloud config set project delta-entry-496910-e7
  3. Enable Vertex AI: gcloud services enable aiplatform.googleapis.com
  4. Get a temporary token: gcloud auth print-access-token
  5. Paste the token into the API Key field.
  6. Use API URL built by the Build Vertex AI URL button and model google/gemini-2.5-flash.
  7. If global fails, try location: us-central1
